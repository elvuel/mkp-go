### What is MKP?

In this repository, **MKP** means a serial-controlled **Mouse/Keyboard recording and playback device protocol**.

An MKP device is usually an external USB-HID capable board or firmware. A host program sends CLI commands such as `m10`, `kpad`, `alog`, and `aplay` to the device over a serial port. The device then emits USB HID mouse/keyboard events to the target computer, or records and replays those events on the device side.

**mkp-go** is a Go SDK / client library for the MKP protocol. It helps you:

- open and manage the MKP serial port;
- send raw MKP CLI commands;
- control mouse and keyboard actions;
- start/stop device-side recording and replay;
- manage log files stored on the device;
- parse JSON/text responses returned by the firmware;
- use higher-level helper/controller APIs for automation tasks.

> In short: **MKP is the hardware/firmware-side mouse-keyboard automation and replay protocol; mkp-go is the Go library for controlling it.**

### Project scope

mkp-go is not just an OS-level mouse/keyboard simulation package. Its main data path is:

```text
Go program -> Serial port -> MKP device/firmware -> USB HID mouse/keyboard events -> Target system
```

It is useful when you need:

- input events generated by an external HID device;
- device-side input recording, storage, inspection, and replay;
- a Go wrapper around low-level MKP CLI commands.

### Features

| Feature | Description | Commands/APIs |
|---|---|---|
| Serial connection | Configure and read/write the device serial port | `NewSFSerialPort`, `Open`, `Read`, `Close` |
| Raw commands | Send arbitrary MKP CLI commands, sync or async | `SendDirective`, `SendDirectiveAsync` |
| Mouse control | Relative move, buttons, wheel, release | `m10`, `Mouse10`, `MouseReleaseAll` |
| Keyboard control | Down/up/tap/combo/release-all | `kpad`, `Keypad`, `helper.KeyTap` |
| Recording | Record input logs on the device | `alog`, `StartRecording`, `helper.Alog` |
| Replay | Replay recorded device-side logs | `aplay`, `StartReplaying` |
| Stop/cancel | Stop recording or cancel replay | `astop`, `acancel` |
| File management | List directories, delete files, clean directories | `list_dir`, `delete_file`, `clean_dir` |
| Device info | Serial number, heartbeat, version, log metadata | `sn`, `alive`, `aversion`, `atime`, `ainsp` |

See the complete command reference: [directives.en.md](./directives.en.md).

### Quick start

```go
package main

import (
    "fmt"

    mkpgo "github.com/elvuel/mkp-go"
    "github.com/elvuel/mkp-go/helper"
)

func main() {
    sfport := mkpgo.NewSFSerialPort()
    sfport.Name = "COM5" // Change to your actual device port.

    if err := sfport.Open(); err != nil {
        panic(err)
    }
    defer sfport.Close()

    // Required when waiting for synchronous command output.
    go sfport.Read()

    sn, err := helper.DeviceSN(sfport)
    if err != nil {
        panic(err)
    }

    fmt.Println("device sn:", sn.SN)
}
```

### Mouse example

```go
// Relative mouse move: x +100, y +50.
err := sfport.Mouse10(mkpgo.NewM10Option().SetX(100).SetY(50))

// Press left button.
err = sfport.Mouse10(mkpgo.NewM10Option().WithLeftButton())

// Release all mouse buttons.
err = sfport.MouseReleaseAll()

// Synchronous send: wait for completion but ignore m10 output.
err = sfport.Mouse10(mkpgo.NewM10Option().SetX(10).WithAsync(false))
```

### Keyboard example

```go
// Tap A.
err := helper.KeyTap(sfport, "A")

// Hold W.
err = helper.KeyDown(sfport, "W")

// Release W.
err = helper.KeyUp(sfport, "W")

// Tap Ctrl+C.
err = sfport.Keypad(
    mkpgo.NewKpadOption().
        WithKeys([]string{"CTRL", "C"}).
        WithAutoRelease(),
)

// Release all keyboard states.
err = helper.KeypadReleaseAll(sfport)
```

### Recording and replay example

```go
// Start recording asynchronously.
err := sfport.StartRecording("demo")

// Stop recording.
err = sfport.StopRecording()

// Replay demo asynchronously with delay 0.
err = sfport.StartReplaying("demo", 0)

// Cancel replay.
err = sfport.CancelReplay()
```

To send `alog` synchronously and wait for output:

```go
out, err := helper.Alog(sfport, "demo", nil)
fmt.Println(out, err)
```

### File and device info examples

```go
fs, err := helper.ListDir(sfport, "/eMMC/applog")
version, err := helper.Aversion(sfport)
hb, err := helper.Alive(sfport)
length, err := helper.Atime(sfport, "demo/record1")
info, err := helper.AInspect(sfport, "demo/record1")
```

### Command modes

mkp-go supports three send modes:

| Mode | API | Description |
|---|---|---|
| Sync with output | `SendDirective` | Wait for device output and return raw output. |
| Sync ignoring output | `SendDirectiveIgnoreOutput` | Wait for completion but return empty output; useful for `m10` and `kpad`. |
| Async | `SendDirectiveAsync` | Write to the serial port without waiting; useful for high-frequency input events. |

### Notes

- For synchronous commands, make sure `SyncOuputEnabled=true` and `go sfport.Read()` is running.
- `m10` and `kpad` are asynchronous by default. Use `WithAsync(false)` when you need to wait for completion.
- File deletion/cleaning helpers are intentionally limited to `/eMMC/applog` to reduce accidental damage.
- Defaults from `NewSFSerialPort()` may need to be adjusted for your actual port, VID/PID, and serial number.
- Firmware support may vary by device/firmware version.
